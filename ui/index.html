How BPE Works

The tokenizer first uses a regex pattern to split text into words (e.g. 'Hello, world!' → ['Hello', ',', ' world', '!']). It then iteratively merges adjacent characters based on a trained frequency list until no more merges can be made. Decoding is simply looking up the IDs and joining the strings.

1. Token Embedding

$$ E \in \mathbb{R}^{V imes d} $$

Each token ID acts as an index to look up a dense continuous vector from the embedding matrix. This vector represents the semantic meaning of the token.

Token ID: 42 → [0.12, -0.45, 0.88, ..., 0.03]

2. Rotary Positional Embedding (RoPE)

$$ R_{\Theta,m}x = (x_1 \cos m heta_1 - x_2 \sin m heta_1, x_1 \sin m heta_1 + x_2 \cos m heta_1, ...) $$

RoPE injects positional information by rotating the Query and Key vectors in a 2D plane based on their position in the sequence.

3. Root Mean Square Normalization (RMSNorm)

$$ ext{RMSNorm}(x) = rac{x}{\sqrt{ rac{1}{d} \sum x_i^2 + \epsilon}} \odot \gamma $$

Unlike LayerNorm, RMSNorm doesn't center the mean to zero, it simply scales the vector by its Root Mean Square. Adjust the input variance below to see the bars normalize!

5. SwiGLU FFN

$$ ext{SwiGLU}(x) = (x W_1 \cdot ext{SiLU}(x W_1)) \odot (x W_3) $$

The Feed-Forward Network splits the input into two branches. One branch applies the non-linear SiLU activation to act as a "gate", controlling how much of the other branch's linear projection passes through via a Hadamard (element-wise) product.

The final vector is projected back to the vocabulary size (1978 tokens). We apply Temperature scaling and Softmax to get a probability distribution, then sample the next token!
