Mathematical Foundations

The exact mathematical formulas defining our LLaMA-based forward pass.

The complete inference step of the network, from token IDs to output logits, can be summarized mathematically by the following consolidated equations:

$$ \begin{aligned} H_0 &= \text{Embed}(X) \\ H_l &= F_l(H_{l-1}) \quad \text{for } l \in \{1, 2, \dots, N\} \\ \text{Logits}(X) &= \text{RMSNorm}_{\text{final}}(H_N) W_{\text{head}}^T \\ F_l(H) &= \tilde{H} + \left[ \text{SiLU}\left( \text{RMSNorm}_{\text{ffn}}(\tilde{H}) W_1^T \right) \odot \left( \text{RMSNorm}_{\text{ffn}}(\tilde{H}) W_3^T \right) \right] W_2^T \\ \text{where} \quad \tilde{H} &= H + A_l\big(\text{RMSNorm}_{\text{attn}}(H)\big) \\ A_l(x) &= \text{softmax}\left(\frac{\text{RoPE}(x W_q^T)\text{RoPE}(x W_k^T)^T}{\sqrt{d_{\text{head}}}} + M\right)(x W_v^T) W_o^T \end{aligned} $$

This formulation ensures stability during deep training and allows for zero-allocation inference during production.

Rotary Positional Embeddings (RoPE)

Unlike absolute positional embeddings that are simply added to the input, RoPE injects positional information by rotating the Query and Key vectors in a complex 2D plane. For a vector pair \((x_1, x_2)\) at position \(m\), the rotation is applied as:

$$ \begin{pmatrix} q_1 \\ q_2 \end{pmatrix} = \begin{pmatrix} \cos(m\theta) & -\sin(m\theta) \\ \sin(m\theta) & \cos(m\theta) \end{pmatrix} \begin{pmatrix} x_1 \\ x_2 \end{pmatrix} $$

This allows the attention dot-product to naturally decay as the relative distance between two tokens increases, giving the model a strong sense of sequence order without hardcoded absolute positions.

SwiGLU Activation Function

Instead of a standard ReLU or GELU, the Feed-Forward Network utilizes a Swish-Gated Linear Unit (SwiGLU). It computes two separate linear projections, applies the SiLU activation \(x \cdot \sigma(x)\) to one, and multiplies them together:

$$ \text{SwiGLU}(x) = \text{SiLU}(x W_1^T) \odot (x W_3^T) $$

This element-wise gating mechanism (\(\odot\)) acts as an information filter, allowing the network to selectively pass relevant features forward. Empirically, this yields significantly better performance in modern LLMs like Llama.

KV Caching (Autoregressive Generation)

During inference, generating the \(N^{\text{th}}\) token requires computing attention over all \(N-1\) previous tokens. To avoid recalculating the Keys and Values for previous tokens, we dynamically cache them in memory:

$$ K_{\text{cache}} = [K_0, K_1, \dots, K_{N-1}] \quad \text{and} \quad V_{\text{cache}} = [V_0, V_1, \dots, V_{N-1}] $$

When the new query \(Q_N\) is generated, it simply attends to the cached matrices, converting a highly expensive \(O(N^2)\) operation into a fast \(O(N)\) operation per generated token.

Top-K Sampling (Logit Truncation)

To prevent the model from occasionally hallucinating highly unlikely "gibberish" tokens, we apply Top-K filtering before the softmax normalization. We sort the logits, keep only the \(K\) highest values, and force all others to \(-\infty\):

$$ v_i = \begin{cases} \text{logit}_i & \text{if } \text{logit}_i \in \text{Top}_K(\text{logits}) \\ -\infty & \text{otherwise} \end{cases} $$

When passed through the softmax function, the \(-\infty\) values collapse to exactly \(0\%\) probability, mathematically guaranteeing that the model only samples from the most sensible candidates.

Technical Implementation

Key components that make TinyLLM efficient and powerful.

  • Rotary Positional Embeddings (RoPE)

    Injects relative positional information directly into the attention mechanism by rotating the query and key representations. It uses a complex frequency spectrum to map positions.

  • Root Mean Square Normalization (RMSNorm)

    Used instead of standard LayerNorm. It removes the mean-centering step for maximum computational efficiency. Applied in a "Pre-Norm" fashion before Attention and FFN blocks.

  • SwiGLU Activation

    The Feed-Forward Network uses the Swish-Gated Linear Unit, utilizing the element-wise Hadamard product to gate information flow intelligently.

  • Byte-Pair Encoding (BPE)

    A custom Python tokenizer that parses raw text and merges adjacent character pairs based on corpus frequency, matching HuggingFace spec.
